# Network Hardening Planner 

Sistema di pianificazione automatica per la messa in sicurezza di infrastrutture di rete.

---

## Idea del Progetto

Le infrastrutture di rete aziendali spesso presentano porte aperte considerate insicure dalle policy di sicurezza moderne: HTTP non cifrato (porta 80), Telnet (porta 23), FTP (porta 21), e altri protocolli legacy. Chiudere queste porte non è un'operazione banale perché servizi attivi potrebbero dipendere da esse, alcuni servizi sono critici per il business, e le dipendenze tra componenti rendono complessa la sequenza di operazioni.

**Network Hardening Planner** risolve questo problema utilizzando tecniche di Intelligenza Artificiale, specificamente **Automated Planning**. Il sistema:

1. Riceve in input la configurazione attuale della rete
2. Riceve le policy di sicurezza (quali porte chiudere)
3. Genera automaticamente un piano ottimale di azioni
4. Minimizza il costo operativo (downtime, riconfigurazioni)

---

## Come Funziona

Il problema è formalizzato come un problema di **Classical Planning**:

```
STATO INIZIALE                        GOAL
┌─────────────────────┐              ┌─────────────────────┐
│ Host: webserver     │              │                     │
│ Porte: 80, 443, 22  │    ====>     │ Porta 80: CHIUSA    │
│ Servizi: http, ssh  │   PLANNER    │ Porta 23: CHIUSA    │
│                     │              │ (su tutti gli host) │
│ Host: database      │              │                     │
│ Porte: 3306, 23     │              │                     │
│ Servizi: mysql      │              │                     │
└─────────────────────┘              └─────────────────────┘
```

Il planner (Fast-Downward) esplora lo spazio degli stati e trova la sequenza di azioni a costo minimo.

---

## Azioni Disponibili

| Azione | Costo | Quando si usa |
|--------|-------|---------------|
| `chiudi_porta` | 1 | Porta senza servizi attivi |
| `disattiva_servizio` | 5 | Servizio non critico da fermare |
| `migra_servizio` | 3 | Spostare servizio su porta alternativa |

**Esempio di piano generato:**
```
1. migra_servizio(webserver, http, 80 → 8080)
2. disattiva_servizio(mailserver, smtp)
3. chiudi_porta(mailserver, 25)
4. chiudi_porta(database, 23)
```
